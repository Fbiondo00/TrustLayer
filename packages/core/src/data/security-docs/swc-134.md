# Message Call with External Data (Arbitrary Call) (SWC-134)

**Severity:** high
**Slither Check:** `arbitrary-lowlevel-call`

## Description

Arbitrary message calls occur when a contract makes a low-level `.call()` to an address and with calldata that are both controlled by an external caller. This gives the caller the ability to execute any function on any contract from the context of the vulnerable contract. The attacker can call functions on other DeFi protocols, transfer tokens held by the contract, approve token spending, or perform any action that the vulnerable contract is authorized to do.

This vulnerability is especially dangerous in the DeFi ecosystem where contracts often hold approvals to spend tokens on other protocols. If a contract has an unlimited approval to spend a user's tokens on Uniswap, and that contract has an arbitrary call vulnerability, an attacker can use the arbitrary call to call `transferFrom` on the token contract, moving the user's tokens to the attacker's address.

The vulnerability often appears in contracts designed to be flexible or "generic executors" — contracts that are intended to execute arbitrary operations on behalf of users. Without proper access control and target whitelisting, these contracts become universal attack vectors. The pattern is also common in governance executors, timelock controllers, and meta-transaction relayers where flexibility is a design goal but security is an afterthought.

## Vulnerable Code

```solidity
function callTarget(address target, bytes calldata data) external payable {
    target.call{value: msg.value}(data);
}
```

## Fixed Code

```solidity
function callTarget(address target, bytes calldata data) external payable onlyOwner {
    require(allowedTargets[target], "Target not allowed");
    (bool ok, ) = target.call{value: msg.value}(data);
    require(ok, "Call failed");
}
```

## Real-World Impact

The **Uniswap V3 exploit** in July 2023 (affecting certain router implementations) involved arbitrary call patterns where an attacker could manipulate the router into making unintended calls. While Uniswap itself was not directly exploited, the pattern of allowing arbitrary calls through router contracts has been a recurring issue in DeFi.

The **Gnosis Safe (now Safe)** proxy architecture was carefully designed to avoid arbitrary call vulnerabilities, but earlier versions of similar multi-sig wallets were not as careful. Several wallet implementations allowed any function to be called on any address from the wallet's context, which could be used to steal approved tokens or manipulate protocol positions.

For AI agent contracts, an arbitrary call vulnerability is among the worst possible issues. AI agents typically have broad permissions across DeFi protocols — they can trade on DEXs, lend and borrow on lending platforms, stake tokens, and manage LP positions. If an attacker can make arbitrary calls from the agent's context, they can perform any of these actions as if they were the agent, potentially draining all user funds and the agent's operational capital in a single transaction.

## How TrustLayer Detects This

TrustLayer's Slither scanner uses the `arbitrary-lowlevel-call` detector to identify functions where both the call target and calldata originate from user-controlled inputs. The detector performs taint analysis to trace the flow of external data through the contract and flags any path where user-controlled data reaches a `.call()`, `.delegatecall()`, or `.staticcall()` instruction.

The Permission Mapper flags this as an **arbitrary_call** risk, one of the most critical negative permissions in the TrustLayer scoring model. The presence of user-controlled arbitrary calls almost always results in an F-level trust score due to the extreme risk. This contributes to the trust score through the Slither weight (30%) and Permissions weight (25%). TrustLayer's AI analysis provides detailed explanations of the risk and recommends implementing target whitelisting, access control, and call data validation.
